3. Coulomb Drift Fix
тД╣я╕П
Both strings are running the drift fix. String-1 has top + bottom anchors. String-2 has top + bottom + experimental mid anchor. The fix is inlined directly in each YAML тАФ not as an external package.

Problem

JK-BMS relies solely on coulomb counting and only recalibrates SoC from voltage when the total_battery_capacity setting is changed. After many cycles the counter drifts тАФ you get readings like 85% when the battery is genuinely full. JK have been asked to fix this for years and refused, instead suggesting purchasing newer hardware.

Fix Mechanism

When anchor conditions are met, the ESPHome lambda toggles the BMS capacity number: 95Ah тЖТ 94Ah (triggers JK recalibration from voltage), then 94Ah тЖТ 95Ah (restores correct value) 10 seconds later via a second interval. No delay() used тАФ prevents watchdog panics.

JK-BMS recalibrates SoC from voltage when capacity is toggled тАФ it does NOT reset to 100%. This means anchoring at top and bottom (steep parts of LFP curve) is safe and accurate. Mid-cycle anchoring is unreliable due to the flat LFP curve between 20тАУ80% SoC.

Anchor Points

AnchorTrigger ConditionsReset ConditionStrings
TOP V тЙе 56.4V AND I тЙд 2A AND SoC тЙе 98% held for 30s (6 ├Ч 5s checks) V < 55.0V String-1 + String-2
BOTTOM V тЙд 48.0V AND I тЙе -2A AND SoC тЙд 5% V > 50.0V String-1 + String-2
MID (experimental) Avg cell V 3.68тАУ3.72V AND |I| тЙд 1A (resting) тАФ fires once per rest event Avg cell V < 3.60V or > 3.80V String-2 only

6. Compile Pipeline
тЪая╕П
Never run esphome compile directly. Always use compile.bat which runs patch_and_compile.ps1. Without the patches the compile either fails or produces a binary from wrong package versions.

Working Directory

C:\Users\osagk\Documents\0_My_Projects\Solar_Project\AI_Assist_Project\

Compile Command

# String-1
compile.bat YamBMS_STRING1_ORIGINAL.yaml

# String-2
compile.bat YamBMS_STRING2_ORIGINAL.yaml

What patch_and_compile.ps1 Does

  1. Pins all remote package refs to ref: 1.5.8 (prevents drift to main/v1.6.0)
  2. Sets refresh: 365d on all packages (prevents ESPHome re-downloading cached packages on every compile)
  3. Patches all cached bms_sensors_JK_BLE_standard.yaml files: pins esphome-jk-bms@2.5.0 (fixes errors_bitmask removal) and injects id: bms${bms_id}_total_capacity_number
  4. Runs git add . && git commit to fix CMake head-ref error during PlatformIO build
  5. Creates head-ref files for all build directories
  6. Runs esphome compile
ЁЯЪи
A Gemini-generated script once ran esphome config first, which re-downloaded packages before patching тАФ overwriting pinned versions and producing a non-functional binary. Never run esphome config before patching.

Key Pinned Versions

PackagePinned toReason
esphome-yambmsref: 1.5.8v1.6.0 has unresolved charge/discharge issue
esphome-jk-bms (syssi)@2.5.0v3.0.0 removed errors_bitmask sensor тАФ breaks compile
refresh365dPrevents ESPHome re-downloading on every compile

Why Local Drift Fix Cannot Be a Remote Package

ESPHome fetches remote packages from GitHub. Local files cannot be included via the files: syntax in a remote package block тАФ it will error with "does not exist in repository". The drift fix is therefore inlined directly as globals: and interval: blocks in each YAML.

TermFull NameDescription
SoCState of ChargeBattery charge level as a percentage (0тАУ100%). Calculated by the BMS coulomb counter тАФ subject to drift over time without anchoring.
SoHState of HealthBattery health as a percentage. Compares current capacity against original rated capacity. 98% = battery is still near new.
OVPOver Voltage ProtectionCell-level protection threshold. BMS cuts charge if any cell exceeds this voltage. Your setting: 4.250тАУ4.260V/cell.
OVPROver Voltage Protection RecoveryVoltage a cell must drop to before charging resumes after OVP triggers.

UVPRUnder Voltage Protection RecoveryVoltage a cell must rise to before discharging resumes after UVP triggers.
OCPOver Current ProtectionBMS cuts off if charge or discharge current exceeds the maximum threshold.
CCLCharge Current LimitMaximum charge current requested by YamBMS to the inverter via CAN bus.
DCLDischarge Current LimitMaximum discharge current requested by YamBMS to the inverter via CAN bus.
CVLCharge Voltage LimitMaximum charge voltage requested by YamBMS to the inverter via CAN bus.
BulkBulk ChargeMain charging phase. Inverter pushes maximum current until pack voltage reaches bulk voltage (57V on String-1, ~55.4V on String-2).
FloatFloat ChargeMaintenance phase after bulk. Inverter holds pack at float voltage (56.6V) with minimal current to keep battery topped up without overcharging.

EOCEnd of ChargeCharge cycle completion. YamBMS EOC timer confirms full charge is complete before switching to float.
Cut-OffCut-Off Charge StatusYamBMS charge status when charging is suspended тАФ e.g. all cells balanced, BMS protection triggered, or v1.6.0 bug.
CAN busController Area NetworkCommunication protocol between YamBMS and the inverter. Uses PYLON V2 protocol. YamBMS sends battery state; inverter sends heartbeat acknowledgement (0x305 frame).
PYLON V2Pylontech Protocol V2CAN bus battery communication protocol used by YamBMS. Recognised by Solis, Sofar and most hybrid inverters.

Coulomb CounterтАФInternal BMS mechanism that tracks SoC by measuring charge in and out (Ah). Drifts over time without a voltage reference reset. The core problem the drift fix solves.
Delta Cell VDelta Cell VoltageDifference between highest and lowest cell voltage in the pack. Indicates cell imbalance. JK-BMS balancing activates when delta exceeds balance trigger voltage (0.010V).
