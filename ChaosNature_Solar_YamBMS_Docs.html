3. Coulomb Drift Fix

Problem

JK-BMS coulomb counter drifts over time, causing SoC to show incorrect values (e.g. 85% when battery is actually full). The BMS needs an anchor event to recalibrate its coulomb counter to 100%.

Root Cause

The JK-BMS recalibrates SoC when the total_battery_capacity number entity is changed. YamBMS does not trigger this automatically.

Solution

New package file packages/bms/bms_soc_drift_fix_JK_BLE.yaml monitors voltage, current and SoC. When all three indicate full charge held for 30 seconds, it toggles the capacity number to force recalibration.

Files Modified

packages/bms/bms_combine_JK_BLE_standard.yamlAdded include for drift fix package

Trigger Conditions (all three for 30 seconds)

4. How to Flash a Device

First time — USB flash

python -m esptool --port COM6 write-flash 0x0 ".esphome\build\yambms-1\.pioenvs\yambms-1\firmware.factory.bin"
⚠️ If the live device is already at the same static IP, disconnect it first to avoid IP conflict during boot.

OTA update (subsequent flashes)

python fix_both.py
python compile_test.py YamBMS_STRING1_TEST.yaml 192.168.0.101

Flash YamBMS-2 (when ready)

python compile_test.py YamBMS_STRING2_TEST.yaml 192.168.0.21

NOTE YamBMS_STRING2_TEST.yaml not yet created. Plan HA entity migration first.

IP Conflict Rule

ESPHome static IP is not enforced by the router. If two devices boot with the same static IP they will fight. Always ensure only one device is powered on per IP.

8. Known Issues

BMS4 Cell 13 low voltage: 3.525V vs others at ~3.58V (59mV delta). Possible bad cell or connection. Monitor closely — may need physical inspection.
Safe mode counter: YamBMS-1 shows "Last reset too quick; invoke in 7 restarts" due to multiple reflashes today. Will trigger safe mode at 10. Will reset itself after a clean uptime period.
Display FAILED: AtomS3 display shows "display is marked FAILED: unspecified" — GPIO33/34 PSRAM conflict. Non-critical, CAN bus and BLE working fine.
Decimal precision: Some sensor values show excessive decimals e.g. -6.08000040054321. Fix: add accuracy_decimals: 2 to sensor configs in future compile.
