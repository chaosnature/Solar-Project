# YamBMS SoC Coulomb Drift Fix
# Developed: June 25 2026 | YamBMS v1.6.0 | ESPHome 2025.11.0 | M5Stack AtomS3

## Problem
JK-BMS coulomb counter drifts over time, causing SoC to show incorrect values
(e.g. 85% when battery is actually full). The BMS needs an anchor event to
recalibrate its coulomb counter to 100%.

## Root Cause
The JK-BMS recalibrates SoC when the total_battery_capacity number entity is
changed. YamBMS does not trigger this automatically.

## Solution
A new package file monitors voltage, current and SoC. When all three indicate
a full charge held for 30 seconds, it toggles the capacity number to force
recalibration.

## Trigger Conditions (all three must be true for 30 seconds)
  - Battery voltage >= 56.4V (just below float voltage of 56.6V)
  - Charge current <= 2A (nearly stopped charging)
  - SoC >= 98%

Reset condition: voltage drops below 55.0V (new discharge cycle started)

## How It Works
When triggered, the fix toggles total_battery_capacity from 95Ah to 94Ah
then back to 95Ah. This change event causes the JK-BMS to recalibrate its
coulomb counter to 100% SoC, correcting any drift.
